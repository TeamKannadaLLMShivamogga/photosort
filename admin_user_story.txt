
# Super Admin User Stories & UI Specifications
**Actor:** Super Admin
**Role:** Platform oversight, revenue tracking, and system configuration.

---

## 1. Page: Global Dashboard (Overview)
**Goal:** Health check of the SaaS platform.

### UI Components & Layout
*   **Header:** "Analytics" + "Super Admin Console" subtitle.
*   **Metric Cards (4):**
    1.  **Platform Revenue:** 
        *   Value: Total of all event prices (₹).
        *   Detail: "+18.4% growth" (Green text with TrendingUp icon).
        *   Visual: CreditCard icon watermark.
    2.  **Platform Balance:**
        *   Value: Outstanding payments (`price` - `paidAmount`).
        *   Style: Dark Slate background, White text.
        *   Detail: "To be collected" (Amber text).
    3.  **Cloud Storage:**
        *   Value: Calculated GB (Photos * 4.5MB).
        *   Visual: Progress bar (Indigo) showing usage vs 2000GB limit.
    4.  **Global Uptime:**
        *   Value: "99.9%".
        *   Visual: Green status dots.
*   **Revenue Analytics Chart:**
    *   **Component:** Area Chart (Recharts).
    *   **Axes:** X = Month (Jan-Jun), Y = Revenue (₹).
    *   **Style:** Indigo gradient fill, tooltip on hover.
*   **Top Partners Sidebar:**
    *   **Header:** "Top Partners".
    *   **List Items:**
        *   Avatar (Square).
        *   Name + Tier (e.g., "Gold").
        *   Revenue Value (Right aligned, bold).
    *   **Interaction:** Hover highlights row.

---

## 2. Page: Global Events
**Goal:** Audit all projects.

### UI Components & Layout
*   **Filter Bar:**
    *   **Search:** Input "Search all events...".
    *   **Status Tabs:** All | Active | Completed.
    *   **View Toggle:** Grid (Card) vs List (Table).
*   **List View (Table):**
    *   **Columns:** 
        1.  **Project Name:** Cover Thumb + Name + Date.
        2.  **Partner:** Name of the Photographer (looked up via `photographerId`).
        3.  **Revenue:** Price (₹).
        4.  **Actions:** "View" button (Indigo Pill).
    *   **Interaction:** Clicking "View" sets Active Event and navigates to `PhotographerEventDetail` (Read-only mode).
*   **Card View:**
    *   Displays standard Event Cards.
    *   **Overlay:** "Project Value" footer.

---

## 3. Page: User Governance
**Goal:** Manage photographer and client accounts.

### UI Components & Layout
*   **Header Stats:** Card showing "Total Users" count.
*   **Search Bar:** Filters table by Name or Email.
*   **User Table:**
    *   **Columns:**
        1.  **User Profile:** Large Avatar + Name + Email.
        2.  **Role:** Colored Badge (Purple=Admin, Indigo=Photographer, Gray=User).
        3.  **Joined:** Formatted Date.
        4.  **Status:** Pill (Green "ACTIVE" / Red "SUSPENDED").
        5.  **Access:** Action Button.
    *   **Action Button:**
        *   If Active: Shows `ToggleRight` icon (Green). Clicking sends `PATCH` request to disable.
        *   If Suspended: Shows `ToggleLeft` icon (Red). Clicking sends `PATCH` request to enable.

---

## 4. Page: Subscriptions
**Goal:** Track studio tiers.

### UI Components & Layout
*   **Filter:** Search by Studio Name.
*   **Stats:** Card showing "ACTIVE STUDIOS" count.
*   **Subscription Table:**
    *   **Filter:** Shows only users with `role == 'PHOTOGRAPHER'`.
    *   **Columns:**
        1.  **Studio:** Avatar + Name + Email.
        2.  **Tier:** Badge (Studio/Pro/Free).
        3.  **Stats (E/P/U):** Compact pill showing Events/Photos/Users counts.
        4.  **Subscription Till:** Expiry Date.
        5.  **Status:** Enabled/Disabled.
        6.  **Access:** Toggle Switch.

---

## 5. Page: System Settings
**Goal:** Global config and database maintenance.

### UI Components & Layout
*   **Config Card (System Engine):**
    *   **Items:** 
        *   "Auto Escalations" (Checkbox).
        *   "Auto-Audit" (Checkbox).
    *   **Visual:** Toggle switches (Green when active).
*   **Danger Zone Card:**
    *   **Style:** Red-50 Background, Red Border.
    *   **Content:**
        *   Title: "Danger Zone" with Alert Triangle.
        *   Warning Text: "Resetting the database will permanently delete all events..."
    *   **Action:** "Reset Database" button (Solid Red).
*   **Reset Confirmation Modal:**
    *   **Trigger:** Clicking "Reset Database".
    *   **UI:** 
        *   Large Red Alert Icon.
        *   "Confirm Reset?" title.
        *   **Buttons:** "Cancel" (Gray), "Yes, Wipe Everything" (Red).
    *   **Logic:** Calls `resetDatabase()` in context -> Clears LocalStorage -> Reloads window.
