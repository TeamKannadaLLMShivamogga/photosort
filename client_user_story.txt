
# Client User Stories & UI Specifications
**Actor:** Client (End User / Bride & Groom)
**Role:** Browse photos, select favorites, review edits, and download.

---

## 1. Page: Event Selector (Landing)
**Goal:** Choose which event to view (if multiple assigned).

### UI Components & Layout
*   **Hero:** "Welcome, [User Name]" with Camera icon.
*   **Search:** Centered Search input "Search events...".
*   **Event Cards Grid:**
    *   **Visual:** Large cover image (16:10 aspect), gradient overlay.
    *   **Details:** Event Name, Date (formatted: Month Year), Photo Count.
    *   **Interaction:** Whole card is clickable -> Navigates to **User Dashboard**.
*   **Empty State:**
    *   Condition: `assignedUsers` list in DB does not contain current user ID.
    *   UI: "No events found" message + "Refresh Data" button.

---

## 2. Page: User Dashboard
**Goal:** Project status and high-level stats.

### UI Components & Layout
*   **Header:** 
    *   Event Name (H1).
    *   Status Pill (e.g., "Selection Open" - Blue, "Submitted" - Amber).
    *   Date Pill (Gray).
*   **Action Bar (Right):**
    *   **"Switch Event"**: White button. returns to Selector.
    *   **"Select Photos"**: Green CTA button with Arrow icon.
        *   *Visibility:* Only visible if `SelectionStatus` == 'open'.
        *   *Action:* Navigates to `Smart Gallery`.
*   **KPI Cards (4):**
    1.  **Total Photos:** Count of all event photos. Icon: Image.
    2.  **Selected:** Count of photos where `isSelected == true`. Icon: CheckCircle.
    3.  **Deadline:** Formatted Date string from `timeline.selectionDeadline`. Icon: Clock.
    4.  **Balance:** Calculated (`price` - `paidAmount`). Icon: Wallet.
*   **Coverage Breakdown:** 
    *   List of sub-events (e.g., "Haldi", "Wedding").
    *   Shows photo count per sub-event.
*   **"Need Help?" Card:**
    *   Visual: Dark gradient background.
    *   Action: "Contact Studio" button (Simulated).

---

## 3. Page: Smart Gallery
**Goal:** Browse and select photos efficiently.

### UI Components & Layout
*   **Header:**
    *   **Main Tabs:** 
        1. **All** (Default)
        2. **Selected** (Filter `isSelected=true`)
        3. **Edited** (Filter `editedUrl` exists)
    *   **Right Actions:** 
        *   View Toggle (Grid/Feed).
        *   **"Submit Selection"**: Green Button. Shows count (e.g., "Submit (150)"). Clicking opens confirmation dialog.
*   **Filter Bar (Only on 'All' Tab):**
    *   **Tabs:** Grid (Reset), AI Best Picks, People, Events, Tags.
    *   **Dynamic Chips:**
        *   *If People Tab:* Horizontal scroll of Face Avatars + Names. Click to toggle filter. Selected chips turn Indigo.
        *   *If Events Tab:* Text buttons for sub-events.
*   **Photo Grid (Grid View):**
    *   **Layout:** Responsive columns (2 to 6 based on screen width).
    *   **Photo Card:**
        *   **AI Badge:** Green "Sparkles" icon in top-left if `isAiPick`.
        *   **Selection State:**
            *   *Unselected:* Hover shows transparent circle.
            *   *Selected:* Top-right shows solid Blue circle with Checkmark. Border turns Blue. Image scales down slightly (98%).
        *   **Download Button:** Bottom-left on hover.
    *   **Interaction:** 
        *   Clicking center -> Opens Lightbox.
        *   Clicking Check/Circle -> Toggles Selection (Optimistic update).
*   **Photo Feed (List View):**
    *   **Layout:** Single column, large images.
    *   **Details:** Shows Metadata (Tags, People names) below image.
    *   **Actions:** Large "Select Photo" button below image.

## 4. Page: My Selections
**Goal:** Review picks and finalize the order.

### UI Components & Layout
*   **Timeline:** Visual progress bar (Open -> Submitted -> Editing -> Review -> Accepted).
*   **Tabs:** "My Selections" | "Edited Pics".

### Tab A: My Selections (Review Mode)
*   **Header:** Title + Description.
*   **Action:** 
    *   **"Submit for edit"**: Green Button (Top Right).
    *   *Logic:* Sets status to `submitted`. Locks the gallery (removes select buttons).
*   **Grid:** 
    *   Shows only `isSelected` photos.
    *   **Hover Action:** Red Trash icon to unselect (remove from list).
*   **Empty State:** Large "CheckCircle" icon with text "Your selection is empty".

### Tab B: Edited Pics (Delivery Mode)
*   **Header:** "Final Deliverables".
*   **Global Actions:**
    *   **"Download All (ZIP)"**: Black Button. (Available if status == `accepted`).
    *   **"Approve All"**: Indigo Button. Marks all pending edits as `approved`.
*   **Grid:** Shows photos with `editedUrl`.
*   **Photo Card:**
    *   **Status Badge:** Top-right (Green Check = Approved, Amber Alert = Changes Requested).
    *   **Review Controls (Below Image):**
        *   **"Approve"**: Button. Sets `reviewStatus` to 'approved'.
        *   **"Comment"**: Button. Toggles input field.
        *   **Comment Input:** Text field + Send button. Adds comment to DB and sets status to 'changes_requested'.
    *   **Comment History:** Scrollable list of previous comments below the input.

---

## 5. Page: Add-ons Store
**Goal:** Purchase extra services.

### UI Components & Layout
*   **Hero:** "Upgrade Your Experience" banner.
*   **Grid:** 2 Columns.
*   **Product Card:**
    *   **Info:** Title (e.g., "Premium Album"), Price (Green text), Description.
    *   **Action Button:**
        *   *Default:* "Request Add-on" (Black, Plus Icon).
        *   *Pending:* "Request Pending" (Amber pill, Clock Icon).
        *   *Approved:* "Request Approved" (Green pill, Check Icon).
        *   *Rejected:* "Request Rejected" (Red pill, X Icon).
    *   **Interaction:** Clicking "Request" triggers `requestAddon` API call and updates UI to Pending.

---

## 6. Page: Album Design (Draft)
**Goal:** View album layout.

### UI Components & Layout
*   **Layout:** Spread view (2 pages side-by-side).
*   **Visuals:** Skeuomorphic book effect (Shadow spine).
*   **Content:**
    *   Left Page: Image 1.
    *   Right Page: Image 2.
    *   (Logic: Groups AI Picks into pairs).
*   **Controls:**
    *   **Left/Right Arrows:** Absolute positioned circles on edges. Navigate spreads.
    *   **Page Numbers:** Bottom corners.
*   **Comments Sidebar:**
    *   **Toggle:** "Comments" button in header.
    *   **Content:** List of comments + Input field to add notes on the specific spread.
*   **Empty State:** If no AI picks, shows "No highlights available yet".
