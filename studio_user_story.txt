
# Studio (Photographer) User Stories & UI Specifications
**Actor:** Photographer / Studio Owner
**Role:** Manage events, upload assets, track payments, and deliver final galleries.

---

## 1. Page: Studio Dashboard
**Goal:** Operational overview of the business.

### UI Components & Layout
*   **Header:** 
    *   Title: "Dashboard".
    *   Action: "New Event" button (Green #10B981, top-right). Clicking opens the **Create Event Wizard**.
*   **Stats Grid (4 Cards):**
    1.  **Total Events:** Icon (Calendar), Count, Blue background/text.
    2.  **Active Events:** Icon (Clock), Count, Green background/text.
    3.  **Total Photos:** Icon (Camera), Count, Indigo background/text.
    4.  **Balance to Collect:** Icon (Wallet), Amount ₹, Amber background/text (Critical for revenue tracking).
*   **Recent Events Section:**
    *   **Header:** "Recent Events" + "View All" link.
    *   **Layout:** Grid of Event Cards (Max 3 latest).
    *   **Card Anatomy:** 
        *   **Cover Image:** 16:9 aspect ratio, zoom on hover.
        *   **Status Badge:** Top-right overlay (e.g., "ACTIVE" in Green, "CLOSED" in Gray).
        *   **Balance Badge:** Bottom-right overlay (e.g., "50% BAL" in Amber) indicating uncollected revenue.
        *   **Info:** Event Name (Truncated), Date (Calendar icon), Photo Count (Camera icon).
        *   **Progress Bar:** Visual line indicating percentage of payment collected.
    *   **Smart Alerts:** 
        *   If `SelectionStatus` is "Submitted": Show flashing "Start Editing" badge.
        *   If `SelectionStatus` is "Editing": Show "Upload Edits" badge.

---

## 2. Page: Events List
**Goal:** Search, filter, and manage all projects.

### UI Components & Layout
*   **Filter Bar (White Card):**
    *   **Search Input:** "Search events..." (Width: sm-56). Filters by Event Name.
    *   **Date Range:** "From Date" and "To Date" date pickers.
    *   **View Toggle:** Segmented control to switch between **Grid** (LayoutGrid icon) and **List** (List icon) views.
*   **Grid View:**
    *   Displays standard Event Cards (same anatomy as Dashboard).
    *   **Alert Badge:** Shows a red notification bubble count if there are pending actions (e.g., pending addon requests).
*   **List View (Table):**
    *   **Columns:** 
        1.  **Project:** Cover thumb + Name + Date.
        2.  **Status:** Colored Pill (Active/Completed).
        3.  **Revenue:** Total Contract Value (₹).
        4.  **Actions:** "Eye" icon button.
    *   **Interaction:** Clicking a row or the Eye icon navigates to `Event Details`.

---

## 3. Page: Event Details (The Command Center)
**Goal:** Deep management of a specific event's data, users, and financials.

### UI Components & Layout
*   **Navigation:** "Back to Events" link (Top left).
*   **Header Card:**
    *   **Visuals:** Large Cover Image thumbnail (Rounded square), Event Name, Date.
    *   **Status Indicator:** "ACTIVE" pill (Green dot).
    *   **Primary Action:** "Manage Photos" button (Indigo, Navigates to `Event Gallery`).
    *   **Meta:** Displays Event ID (last 6 chars).

### A. Financials & Payments Card
*   **Header:** "Financials & Payments" + Status Badge (e.g., "Pending Balance" in Amber / "Paid in Full" in Green).
*   **Overview Grid:** Three columns:
    1.  **Total Contract:** ₹ Amount (Black).
    2.  **Received:** ₹ Amount (Green).
    3.  **Balance Due:** ₹ Amount (Amber).
*   **Actions:**
    *   **"Record Payment" Button:** Opens **Payment Modal**.
    *   **"View History" Toggle:** Expands/Collapses the transaction history table.
*   **Transaction Table (Collapsible):**
    *   Columns: Date, Method, Note, Amount.
    *   Empty State: "No payments recorded yet."
*   **Payment Modal:**
    *   **Fields:** Amount (Number), Date (Date Picker), Method (Dropdown: UPI, Cash, Bank Transfer, Cheque), Note (Text Input).
    *   **Actions:** "Cancel" (Gray), "Save Record" (Green).

### B. Workflow Status Card
*   **Visual Timeline:** Horizontal progress bar with nodes: Open -> Submitted -> Editing -> Review -> Accepted.
*   **Current Status:** The active node is highlighted (Scale 1.25x, Colored). Past nodes are Green. Future nodes are Gray.
*   **Dynamic Action Buttons (Bottom Right):**
    *   *State: Open* -> **"Force Lock Selection"** (Lock Icon). Sets status to Submitted.
    *   *State: Submitted* -> **"Start Editing Phase"** (Indigo). Sets status to Editing.
    *   *State: Editing* -> **"Submit for Review"** (Indigo). Sets status to Review.
    *   *State: Review* -> **"Re-open Selection"** (Unlock Icon). Reverts to Open.

### C. Client Contact Details Card
*   **Header:** "Client Contact Details" + **"+ Add Client"** toggle button.
*   **User List:**
    *   Rows showing: Avatar, Name, Email (Truncated), Phone.
    *   **Action:** "Trash" icon (Red on hover) to remove user access from this event.
*   **Add Client Form (Slide Down):**
    *   **Inputs:** Name, Email Address, Phone.
    *   **Action:** Green "Plus" button.
    *   **Logic:** If email exists in DB, links existing user. If not, creates new user account and links it.

### D. Event Schedule Card
*   **Header:** "Event Schedule" + Count of sub-events.
*   **List:**
    *   Rows showing: Icon (Calendar), Event Name, Date, Location.
    *   **Action:** "Trash" icon to delete sub-event.
*   **Add Form:**
    *   **Inputs:** Name (e.g., Sangeet), Date, Venue.
    *   **Action:** Black "Plus" button.

### E. Quick Actions Card (Sidebar)
*   **Style:** Dark background (Slate-900), White text.
*   **Links:**
    *   "Send Selection Reminder" (Simulated action).
    *   "Generate Invoice" (Simulated action).
    *   "Delete Event" (Red text, Hover: Red-200).

## 4. Page: Event Gallery (Manage Photos)
**Goal:** Upload, organize, and curate photos.

### UI Components & Layout
*   **Header:** Event Name + Tab Navigation (All | Selected | Edited).
*   **Controls (Right Side):**
    *   **View Toggle:** Grid vs Feed.
    *   **"Upload Raw"**: Black button. Opens file picker. Uploads to general pool.
    *   **"Upload Edits"**: Indigo button. Opens file picker. Uploads and attempts to match filenames to originals.
*   **Sub-Tabs (Filters) - Visible on 'All' Tab:**
    *   **"All Photos"**: No filter.
    *   **"AI Best Picks"**: Shows only photos with `isAiPick = true`.
    *   **"People"**: Shows horizontal scroll of detected faces.
        *   **Interaction:** Clicking a face filters the grid.
        *   **Renaming:** Pencil icon on face chip allows renaming "Unknown" to real name (updates all photos with that face ID).
    *   **"Events"**: Filter by sub-event category (e.g., Haldi).
    *   **"Tags"**: Filter by AI tags (e.g., Decor, Candid).
*   **Photo Grid:**
    *   **Card:** Image aspect ratio 4:5.
    *   **Overlays (Hover):**
        *   **Top Right:** Trash Icon (Delete).
        *   **Bottom Right:** Selection Checkmark (Read-only view of client selection).
        *   **Top Left:** "Sparkles" icon if AI Best Pick.
    *   **Interaction:** Clicking image opens Lightbox.
*   **Status Modal:** Button "Status: [State]" opens modal to update Workflow Status and Delivery Date manually.

---

## 5. Page: Portfolio & Settings
**Goal:** Branding and profile management.

### UI Components & Layout
*   **Header:** Title + "Save Changes" button.
*   **Profile Card:** 
    *   Large Avatar (Click camera icon to upload new).
    *   Studio Name display.
*   **Bio Section:** Textarea for studio description.
*   **Featured Videos:** 
    *   Input field for YouTube URL + "Plus" button.
    *   List of added videos (Thumbnail view). Trash icon to remove.
*   **Sample Gallery:** 
    *   Grid of uploaded portfolio images (Max 6 preview).
    *   **"Upload Photos"**: Button with Loader state.
    *   **Interaction:** Click image to remove (Trash icon overlay).
*   **Service Menu:**
    *   **Add Form:** Inputs for Service Name, Price, Type (Dropdown: Core Service / Add-on). "Plus" button to add.
    *   **Service List:** Cards showing Name, Price, Type badge (Purple for Service, Amber for Add-on).
    *   **Actions:** Edit (Pencil), Delete (Trash).
